# 攻略详情页资料站型排版设计

## 目标

将攻略详情页升级为适合大型资料库的长文阅读布局。采用方案 C：目录导航、完整章节、材料清单和重点提示并存，让文章标题、章节标题和正文内容一眼可辨。

## 内容原则

- 六篇现有攻略的完整正文全部保留，不缩写章节内容。
- 页面中的目录只用于导航，不替代正文。
- 每个二级标题下继续显示该章节全部段落。
- 标题、正文、列表、引用和提示信息使用不同的视觉层级。
- 后台仍使用 Markdown 编辑，不增加新的必填字段。

## 页面结构

1. 顶部主题区显示版本、文章标题、摘要和阅读信息。
2. 正文区域采用左右布局：
   - 左侧为粘性目录，自动收集正文中的二级标题。
   - 右侧为完整攻略文章。
3. 文章开头显示“攻略速览”，包含版本、章节数和预计阅读时长。
4. 二级标题设计为浅金色章节卡，使用绿色标记强调层级。
5. 普通段落保持舒适行宽和行距，段落之间有明确留白。
6. Markdown 列表显示为独立清单；引用内容显示为黄色重点提示框。
7. 表格、图片和代码块保持响应式，不溢出正文容器。

## 交互

- 点击目录项平滑滚动到对应章节。
- 当前章节对应的目录项随滚动高亮。
- 桌面端目录保持在视口中。
- 窄屏设备取消左右布局，目录放在正文上方并允许展开或收起。
- 标题锚点使用稳定且唯一的编号，避免中文标题编码造成跳转问题。

## 实现方式

- 后端继续使用 `marked` 和 `sanitize-html` 输出安全 HTML。
- 前端收到文章 HTML 后，为二级标题补充唯一 `id`，据此生成目录。
- 阅读时长由纯文本字数在前端估算。
- 不修改文章数据库结构，也不要求管理员手工维护目录。
- 样式集中写入 `public/css/app.css`，详情页结构写入 `public/js/app.js`。

## 验证

- API 测试确认详情接口保留完整正文和章节。
- 前端单元测试确认目录能从多个二级标题生成。
- 六篇内置攻略均应至少显示四个目录项。
- 验证桌面端双栏、移动端单栏、长段落、列表和引用块。
- 运行全部现有测试，确保前台、后台和数据接口不受影响。
