# 星露谷物语中文工具产品设计规格

## 1. 目标

在现有星露谷物语中文攻略站中新增三个可独立使用、彼此联动的工具：

1. 鱼类条件查询器
2. 作物收益计算器
3. 社区中心进度清单

三个工具继续使用现有 HTML、Tailwind CSS、原生 JavaScript、Node.js、
Express 和 SQLite 技术栈，复用现有田园像素视觉语言及本地游戏素材。
内容基准为游戏正式版 1.6.15。

首版不增加玩家注册和云同步。社区中心进度保存在当前浏览器，并提供
JSON 导入、导出及重置功能。工具数据由服务端只读 API 提供，计算和
进度逻辑拆分成可独立测试的模块。

## 2. 项目拆分与交付顺序

三个工具属于相对独立的产品子系统，按以下顺序交付：

1. 公共工具数据层和导航入口
2. 鱼类条件查询器
3. 作物收益计算器
4. 社区中心进度清单
5. 跨工具联动、响应式联调、Docker 和上线文档

每个子系统完成后都必须具备独立可访问路由、自动化测试和空状态/错误状态，
不得依赖下一个子系统才能工作。

## 3. 总体架构

### 3.1 服务端

新增 `src/features/tools` 功能边界：

- `data/`：1.6.15 工具专用结构化数据。
- `fish.js`：鱼类筛选与当前条件匹配。
- `crops.js`：种植周期、收获次数和收益计算。
- `bundles.js`：社区中心房间、收集包和物品定义。
- `router.js`：只读工具 API。

工具数据不直接依赖现有 `dataset_entries.attributes_json`。现有资料条目字段
粒度不足以支持严谨计算，工具使用独立、明确、可校验的数据结构；条目名称和
图片路径尽量与资料库保持一致，以便互相跳转。

首版 API：

- `GET /api/tools/fish`
- `GET /api/tools/crops`
- `POST /api/tools/crops/calculate`
- `GET /api/tools/community-center`

筛选和计算参数必须经过 Zod 校验。错误继续使用项目现有统一格式：

```json
{
  "error": {
    "code": "INVALID_TOOL_INPUT",
    "message": "输入条件不正确",
    "details": {}
  }
}
```

### 3.2 前端

新增独立模块，避免继续扩张 `public/js/app.js`：

- `public/js/tools/fish-tool.js`
- `public/js/tools/crop-tool.js`
- `public/js/tools/community-center-tool.js`
- `public/js/tools/community-progress.js`
- `public/js/tools/tool-shell.js`

主路由增加：

- `#tools`
- `#tools/fish`
- `#tools/crops`
- `#tools/community-center`

首页统计区或首页内容区增加“实用工具”入口，顶部导航增加“工具”菜单。
工具页共用标题区、说明区、输入面板、结果面板和版本提示组件。

### 3.3 数据版本

所有工具 API 返回：

```json
{
  "gameVersion": "1.6.15",
  "dataVersion": 1
}
```

社区中心本地进度也保存 `dataVersion`。未来收集包数据变更时，迁移函数应
保留仍存在的物品 ID，并忽略已经删除的 ID。

## 4. 鱼类条件查询器

### 4.1 用户目标

玩家能够回答：

- 当前季节、天气、时间和地点能钓到什么？
- 某条鱼在什么条件下出现？
- 哪些鱼仍需要用于社区中心？
- 当前筛选中哪些鱼是特殊地点、蟹笼或传奇鱼？

### 4.2 数据模型

每条鱼至少包含：

```js
{
  id: "catfish",
  name: "鲶鱼",
  aliases: ["Catfish"],
  image: "/assets/game/36px-Catfish.png",
  seasons: ["春季", "秋季"],
  locations: ["小镇河流", "森林河流"],
  weather: ["雨天"],
  timeRanges: [{ start: 600, end: 2400 }],
  sourceType: "钓竿",
  difficulty: 75,
  behavior: "混合型",
  basePrice: 200,
  bundleIds: ["river-fish-bundle"],
  notes: "秘密森林在夏季雨天也可捕获。",
  gameVersion: "1.6.15"
}
```

时间使用 24 小时整数 `HHMM`，`2400` 表示午夜。全天鱼使用
`[{ start: 600, end: 2600 }]`，跨午夜区间允许 `end > 2400`。

鱼类范围覆盖：

- 普通钓竿鱼
- 特殊地点鱼
- 传奇鱼及传奇鱼二代
- 夜市潜艇鱼
- 蟹笼收获物

蟹笼收获物不参与天气和时间筛选，但可通过来源类型筛选。

### 4.3 筛选条件

- 名称关键词
- 季节：春、夏、秋、冬、全部
- 天气：晴天、雨天、任意
- 时间：玩家输入游戏时间或选择“全天”
- 地点
- 来源：钓竿、蟹笼、特殊活动
- 用途：社区中心、非社区中心
- 分类：普通、传奇、特殊

“此刻能钓什么”模式要求季节、天气和时间，地点可选。匹配规则：

- `全部季节` 匹配任何季节。
- `任意天气` 匹配晴天和雨天。
- 时间落在任一时间段中即匹配。
- 未选择地点时返回所有地点并按地点分组。

### 4.4 页面

桌面端左侧为条件面板，右侧顶部显示命中数量、当前条件摘要和重置按钮；
结果使用紧凑卡片，展示图标、名称、地点、时间、天气、难度和社区中心标记。

移动端筛选区位于结果上方，可折叠。结果改为单列卡片，不使用需要横向滚动的
宽表格。

无结果时展示最可能冲突的条件，并提供“一键放宽天气”或“一键清除地点”。

## 5. 作物收益计算器

### 5.1 用户目标

玩家输入当前规划条件后，能够比较可种作物的：

- 可收获次数
- 种子总成本
- 基础销售收入
- 加工收入
- 净利润
- 每格利润
- 每日平均利润
- 投资回报率

### 5.2 输入

- 季节
- 当前日期，范围 1 至 28
- 地块数量，范围 1 至 9999
- 可用资金，可留空
- 生长加速：无、生长激素、高级生长激素、顶级生长激素
- 农夫技能：无、农业学家
- 售价职业：无、农耕人
- 收益方式：直接出售、罐头瓶、小桶
- 是否拥有温室/姜岛农场
- 是否计入首次购买种子的成本

资金存在时，实际种植格数取地块上限和可购买种子数量的较小值。

### 5.3 数据模型

```js
{
  id: "blueberry",
  name: "蓝莓",
  image: "/assets/game/36px-Blueberry.png",
  seasons: ["夏季"],
  seedPrice: 80,
  seedSource: "皮埃尔杂货店",
  growthStages: [1, 2, 2, 2, 2, 4],
  regrowDays: 4,
  harvestYield: 3,
  extraYieldChance: 0.02,
  baseSellPrice: 50,
  processing: {
    keg: { product: "蓝莓果酒", formula: "fruit-wine" },
    jar: { product: "蓝莓果酱", formula: "jelly" }
  },
  restrictions: [],
  gameVersion: "1.6.15"
}
```

首版覆盖全部可种植并可销售的常规作物、1.6 新增四季种子、温室/姜岛常用
作物及特殊作物。花卉蜜蜂房收益不纳入首版，以避免把不同生产系统混入单格
作物收益。

### 5.4 计算规则

生长速度缩减率：

- 无：`0%`
- 生长激素：`10%`
- 高级生长激素：`25%`
- 顶级生长激素：`33%`
- 农业学家：在肥料缩减率上额外增加 `10%`

肥料与农业学家的缩减率相加，例如高级生长激素与农业学家合计 `35%`。
计算器必须使用每种作物的 `growthStages`，实现与游戏
`HoeDirt.applySpeedIncreases` 一致的逐阶段缩减和取整规则。所有支持的作物
必须提供阶段数据，不允许退回到总天数比例近似。实现测试需使用官方 Wiki
列出的多个作物加速结果作为交叉校验。生长加速只影响第一次成熟，不缩短重复
收获作物的 `regrowDays`。

普通作物：

```text
首次收获日 = 当前日期 + 实际生长天数
首次收获日 > 28 时收获次数为 0
否则可收获次数 = 1 + floor((28 - 首次收获日) / 实际生长天数)
```

重复收获作物：

```text
首次收获日 = 当前日期 + 实际生长天数
首次收获日 > 28 时收获次数为 0
否则收获次数 = 1 + floor((28 - 首次收获日) / 重生天数)
```

温室或姜岛模式不受季末限制，用户改为输入规划天数，范围 1 至 365。

直接出售收入按基础品质计算，并在结果明确标注“不含品质提升”。农耕人职业
按适用产品增加 10%。加工公式：

- 果酒：基础果实售价 × 3
- 果酱：基础果实售价 × 2 + 50
- 果汁：基础蔬菜售价 × 2.25
- 腌菜：基础蔬菜售价 × 2 + 50

每次收获数量为 `种植格数 × harvestYield`。额外产出使用期望值计算并标记
为“平均预期”，不伪装成必然收益。

```text
总成本 = 实际种植格数 × 种子价格 × 需要购买的种植轮数
总收入 = 总产量 × 单位收益
净利润 = 总收入 - 总成本
每格利润 = 净利润 / 实际种植格数
日均利润 = 净利润 / 规划天数
投资回报率 = 净利润 / 总成本
```

无法购买、赶不上成熟、加工方式不适用和资金不足均返回明确原因，不返回误导性
的零利润排名。

### 5.5 页面

顶部为规划条件，提交后生成按“净利润”默认排序的比较表。桌面端同时展示：

- 最佳净利润
- 最佳日均收益
- 最低启动资金

每个作物结果可展开查看计算过程。移动端表格改成卡片，重点数字保持首屏可见。
允许在净利润、日均利润、启动成本和收获次数之间切换排序。

## 6. 社区中心进度清单

### 6.1 用户目标

玩家可以按房间和收集包勾选物品，查看总进度、房间进度、缺失物品及下一季
优先事项，并能备份或迁移自己的进度。

### 6.2 数据模型

```js
{
  id: "pantry",
  name: "茶水间",
  image: "/assets/game/24px-Bundle_Green.png",
  reward: "温室",
  bundles: [
    {
      id: "spring-crops-bundle",
      name: "春季作物收集包",
      reward: "20 个生长激素",
      requiredCount: 4,
      items: [
        {
          id: "parsnip",
          name: "防风草",
          quantity: 1,
          quality: null,
          image: "/assets/game/24px-Parsnip.png",
          seasons: ["春季"],
          source: "种植"
        }
      ]
    }
  ]
}
```

覆盖标准社区中心全部房间和收集包，包括公告栏、金库与废弃 Joja 超市收集包。
可选择项目使用 `requiredCount` 表示完成所需数量，不能要求玩家把全部可选项
都勾选。

### 6.3 本地持久化

LocalStorage 键：

```text
pixelharvest.community-center.v1
```

数据格式：

```json
{
  "schemaVersion": 1,
  "dataVersion": 1,
  "updatedAt": "2026-06-12T12:00:00.000Z",
  "completedItemIds": [
    "spring-crops-bundle:parsnip"
  ]
}
```

保存规则：

- 每次勾选立即保存。
- 页面加载时自动恢复。
- 非法 JSON、错误 schema 或未知 ID 不得使页面崩溃。
- 未知 ID 在迁移时忽略。
- 多标签页通过 `storage` 事件同步。

导出文件名：

```text
pixelharvest-community-center-YYYY-MM-DD.json
```

导入前校验 schema，展示将恢复的完成数，用户确认后覆盖当前进度。重置必须二次
确认。

### 6.4 进度规则

- 单个收集包：已勾选数量达到 `requiredCount` 时完成。
- 房间：房间内全部收集包完成时完成。
- 总进度：以“完成的必需物品槽位 / 全部必需物品槽位”计算。
- 金库以四个捐款项目作为四个可勾选槽位。

页面顶部展示总百分比、已完成收集包数、已完成房间数。筛选包括：

- 全部
- 未完成
- 当前季节可获得
- 即将错过
- 按房间

“即将错过”指当前季节可获得且尚未完成、下季无法获得的物品。首版由用户手动
选择当前季节，不读取真实日期。

## 7. 跨工具联动

- 鱼类卡片中的社区中心标记链接到社区中心清单，并定位相应收集包。
- 社区中心鱼类物品链接到鱼类查询器，并自动带入鱼名。
- 社区中心作物物品链接到作物计算器，并自动选择对应作物和季节。
- 工具结果可链接回现有资料库条目，但资料条目不存在时不显示链接。

联动通过 URL 查询参数实现，不引入全局客户端状态库。

## 8. 视觉与响应式

继续使用现有森林绿、木棕、麦穗金和羊皮纸色板：

- 工具首页使用三张功能卡，不使用大面积新背景图。
- 输入控件保持高对比度和明确标签。
- 结果中的关键数字使用深绿色，警告使用赭红色，不只依赖颜色表达状态。
- 图标保持 `image-rendering: pixelated`。
- 所有按钮和复选框具备键盘焦点样式。
- 390 px、768 px 和 1440 px 宽度均可完成全部核心操作。
- 尊重 `prefers-reduced-motion`。

## 9. 错误处理

- API 加载失败：保留用户输入并显示重试按钮。
- 无匹配结果：说明冲突条件并提供放宽入口。
- 计算输入非法：在对应字段显示错误，不清空其他输入。
- 本地存储不可用：清单仍可在当前页面使用，并提示关闭页面后不会保存。
- 导入文件非法：不修改现有进度。
- 缺失图片：沿用全站默认素材。

## 10. 测试

### 10.1 单元测试

鱼类：

- 多季节、任意天气和多个时间段匹配。
- 时间边界和跨午夜时间段。
- 蟹笼项目不受天气时间限制。
- 多条件交集和无结果。

作物：

- 普通作物最后可种日期。
- 重复收获次数。
- 多轮补种种子成本。
- 资金和地块共同限制。
- 加速肥与农业学家。
- 果酒、果酱、果汁和腌菜公式。
- 额外产出的期望值。
- 温室/姜岛规划天数。

社区中心：

- 可选收集包的 `requiredCount`。
- 房间和总进度。
- 本地数据序列化、反序列化和迁移。
- 非法导入不覆盖现有进度。

### 10.2 API 测试

- 四个工具 API 返回 1.6.15 数据版本。
- 作物计算接口拒绝非法日期、负数资金和未知枚举。
- 鱼类接口组合筛选正确。
- 社区中心返回完整房间和收集包。

### 10.3 页面验收

- 三个路由可直接打开和刷新。
- 桌面、平板、手机布局可操作。
- 导航、返回、跨工具链接正常。
- 进度刷新后仍存在。
- 导出后清空，再导入可以恢复。
- Docker 重建后工具数据和页面仍可访问。

## 12. 非目标

首版不包含：

- 玩家账号和云同步
- 支付或会员权限
- 存档文件解析
- 农场布局编辑器
- 蜂蜜、动物、鱼塘和整套加工链综合模拟
- 模组数据
- 1.7 未正式发布内容

这些能力可以在三个基础工具稳定并获得真实使用数据后单独立项。
